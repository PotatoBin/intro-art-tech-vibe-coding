이 아이디어를 예술공학개론 수업의 브라우저 기반 웹 게임 `PLAN.md`로 구체화하라.

과제 조건:
- AI 에이전트와 바이브 코딩을 사용해 브라우저 기반 웹 게임 만들기
- Vercel로 배포된 랜딩 페이지 URL
- 랜딩 페이지에는 플레이 가능한 게임과 아티스트 스테이트먼트가 모두 포함되어야 함
- 방문자는 랜딩 페이지에서 게임을 바로 플레이하거나 접근할 수 있어야 함
- 장르, 그래픽, 조작 방식에는 제한 없음
- 일반적으로 게임이 아닌 것을 플레이 가능한 경험으로 바꾸는 게임화된 경험도 가능

권장 사항:
- 기술만 있는 프로젝트나 보기만 예쁜 프로젝트로 만들지 말 것.
- 게임에는 명확한 규칙, 상호작용, 이야기, 질문, 시각적 접근, 또는 경험이 있어야 함.
- 익숙한 것을 바탕으로 한다면, 사람들이 그것을 다르게 보고 플레이하게 만들 것.

기획이 아직 불명확하다면 바로 `PLAN.md`를 쓰지 말고, 필요한 확인 질문을 최대 5개까지 먼저 하라.
기획이 충분히 가능하다면 아래 형식으로 `PLAN.md` 내용을 작성하라.

# PLAN.md

## 게임 개요
- 게임 제목
- 한 문장 콘셉트

## 과제 요구사항
- 브라우저에서 실행되는 플레이 가능한 웹 게임
- Vercel로 배포된 랜딩 페이지 URL
- 같은 랜딩 페이지 안에 게임, 짧은 플레이 방법, 아티스트 스테이트먼트 포함

## 플레이어 목표
- 그 목표가 게임 경험 안에서 중요한 이유

## 핵심 반복 구조
- 플레이어가 반복해서 하는 행동
- 게임이 그 행동에 반응하는 방식
- 반복 구조가 긴장감, 발견, 놀이, 의미를 만드는 방식

## 입력과 조작
- 키보드, 마우스, 터치, 또는 기타 조작 방식
- 최소 기능 버전에 필요한 최소 조작
- 필요하다면 접근성 또는 모바일 고려사항

## 규칙과 피드백
- 핵심 규칙
- 점수, 진행도, 시간 제한, 생명, 자원, 또는 기타 상태
- 시각, 소리, 텍스트, 움직임 피드백

## 승리, 실패, 종료 상태
- 승리 조건이 있다면 명시
- 실패 조건이 있다면 명시
- 엔딩, 반복, 성찰, 또는 완료 상태

## 진행과 난이도
- 시간이 지나며 게임이 달라지는 방식
- 첫 버전에서 단순하게 유지해야 할 것

## 시각과 상호작용 방향
- 시각적 분위기
- 인터페이스 배치
- 시각 스타일이 콘셉트를 뒷받침하는 방식

## 아티스트 스테이트먼트 방향
- 무엇을 만들었는가
- 사람들이 어떻게 플레이하는가
- 어떤 아이디어, 이야기, 감정, 질문, 또는 경험이 바탕에 있는가
- AI 또는 에이전트를 어떻게 사용했는가

## 랜딩 페이지 구조
- 제목 또는 첫 화면 영역
- 플레이 가능한 게임 영역
- 짧은 플레이 방법
- 아티스트 스테이트먼트
- 필요하다면 크레딧 또는 에셋 출처

## 기술 스택과 배포 가정
- 추천 프레임워크와 이유
- 렌더링 방식: HTML/CSS, Canvas, Phaser, PixiJS, Three.js, 또는 다른 선택지
- Vercel 배포 가정
- 백엔드, 데이터베이스, 로그인, 리더보드, 저장 기능은 꼭 필요할 때만 포함

## 작업 순서
1. 프로젝트 초기 설정과 랜딩 페이지 기본 구조
2. 가장 작은 플레이 가능한 게임 반복 구조
3. 조작 방식과 피드백
4. 아티스트 스테이트먼트와 제출용 내용
5. 시각적 다듬기와 반응형 레이아웃
6. 빌드, 배포, 최종 점검

## 최소 기능 버전 범위
- 반드시 만들 것
- 가능하면 추가할 것
- 지금은 제외할 것

## 위험 요소와 점검 항목
- 플레이 가능성 관련 위험
- 창의성 관련 위험
- 의미와 아티스트 스테이트먼트 관련 위험
- 요구사항 또는 배포 관련 위험
- 에셋, 저작권, 개인정보, API 키 관련 위험
