# 프로젝트 개요
- 프로젝트 제목
- 한 문장 요약
- 타깃 사용자 또는 관람자
- 핵심 문제, 경험, 의도
- 왜 예술공학 프로젝트에 적합한지

# 과제 유형과 제출 방식
- Option 1 또는 Option 2 중 어디에 해당하는지
- Option 1이라면 결과물 포맷과 프로젝트 소개 랜딩 페이지 구성
- Option 1의 결과물이 웹에서 직접 체험하기 어렵다면 필요한 데모 영상, 스크린샷, 설명
- Option 2라면 웹에서 직접 접근 가능한 작품과 같은 페이지/사이트에 포함할 아티스트 스테이트먼트

# 목표와 비목표
- 반드시 달성해야 하는 것
- 시도하지 말아야 하는 것
- 작고 단순하게 유지해야 하는 것

# 기술 스택
- 프레임워크
- 스타일링 방식
- 주요 라이브러리
- 데이터 저장 방식, 필요한 경우
- 배포 대상
- 이 기술 스택을 선택한 이유

# 핵심 기능
- 필수 기능
- 선택 기능
- 지금은 피해야 할 기능

# 사용자 흐름
- 첫 화면
- 주요 상호작용
- 결과 또는 피드백
- 종료 또는 완료 상태

# 디자인 방향
- 시각적 톤
- 색상 방향
- 타이포그래피 방향
- 레이아웃 원칙
- 상호작용 스타일

# 주요 결정 사항
- 이미 결정된 중요한 사항
- 그 결정을 내린 이유
- 명확한 이유 없이 바꾸지 말아야 할 결정

# MVP 완료 기준
- 프로젝트가 완성되었다고 볼 수 있는 기준
- 반드시 필요한 화면 또는 섹션
- 반드시 필요한 상호작용
- 첫 번째 버전에서 허용되는 한계

# 파일 구조
- 예상되는 주요 폴더와 파일
- 각 중요한 파일의 역할
