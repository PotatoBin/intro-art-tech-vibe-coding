Please read the existing project documentation and start implementing the MVP for this Art & Technology project.

Use the project documentation and current workspace context as the source of truth.
Do not change the core concept, tech stack, design direction, or project direction unless there is a clear blocker.
If the documentation is missing or unclear, state what is missing before making assumptions.

Before coding, do the following:

1. Project Understanding
- Summarize what this project is.
- Explain the Art & Technology concept.
- Identify the intended user, viewer, or participant.

2. MVP Scope
- List the features that must be implemented first.
- Identify what should not be built yet.
- Confirm the page or explanation elements needed for submission.

3. Implementation Plan
- Break the first implementation work into small steps.
- Start with the simplest working version.
- Each step should be possible to complete and verify.

4. Risks
- Point out anything unclear in the documentation.
- Point out any technical or design risks.
- Mention any risk of exposing sensitive information or using external assets without permission.

Then begin implementation.

Work in small steps.
After each major step, explain the changed files, how to verify the result, and what should be done next.

Avoid:
- Adding major new features
- Overengineering
- Changing the core concept
- Making the result only visually decorative
- Making the result only technically impressive without experience, meaning, or interaction
