Vercel CLI를 사용해 이 브라우저 기반 웹 게임을 프로덕션으로 배포하라.

현재 프로젝트 파일과 작업 컨텍스트를 기준으로 진행하라.
배포에 꼭 필요한 수정만 하고, 게임 콘셉트나 주요 구현은 바꾸지 말 것.

배포 전 최소 확인:
- 랜딩 페이지에서 게임을 바로 플레이하거나 접근할 수 있는지 확인
- 랜딩 페이지에 아티스트 스테이트먼트가 보이는지 확인
- API 키, 토큰, 비밀번호, 개인정보, `.env` 파일이 공개되지 않는지 확인
- 빌드 명령이 있다면 실행하고, 실패하면 배포에 필요한 범위 안에서만 수정

Vercel 배포:
- Vercel CLI 사용 가능 여부 확인
- 설치되어 있지 않다면 적절한 패키지 매니저로 설치
- `vercel --prod`로 프로덕션 배포
- 로그인, 프로젝트 연결, 팀 선택, 환경 변수 입력이 필요하면 필요한 입력을 명확히 요청

배포 후 확인:
- 배포된 Vercel URL 제시
- 배포 URL이 실제로 열리는지 확인
- 배포 환경에서 게임이 플레이 가능한지 확인
- 아티스트 스테이트먼트가 보이는지 확인
- 남은 오류나 제출 전 주의사항이 있다면 짧게 정리

Vercel URL이 확인되지 않았다면 배포가 성공했다고 말하지 말 것.
